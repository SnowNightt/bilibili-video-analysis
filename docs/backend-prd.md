# 省流看后端开发 PRD

## 1. 产品背景

省流看是一个本地运行的 Bilibili 视频分析工具。用户输入公开 Bilibili 视频的 BV 号或视频地址后，可以读取视频信息，选择分析模式、分析深度和模型配置，并生成结构化视频分析报告。

当前前端已经具备视频信息读取、模型配置表单、任务进度页、报告页和历史记录页。后端需要补齐本地分析服务能力，负责模型连接验证、模型密钥安全处理、分析任务编排、媒体处理、模型调用、报告生成、报告追问和资源清理。

## 2. 产品目标

- 让用户可以在本地完成一次真实 Bilibili 公开视频分析，而不是依赖前端演示数据。
- 让前端通过统一的 `/api` 后端接口创建任务、查询进度、取消任务、读取报告和继续追问。
- 让用户的模型 API Key 不写入浏览器本地存储，也不出现在日志、错误响应或报告内容中。
- 让长耗时分析任务具备可追踪状态、可取消能力和清晰失败原因。
- 让生成结果以结构化报告形式返回，支持前端展示、TXT 导出和浏览器打印导出 PDF。

## 3. 目标用户与核心场景

### 3.1 目标用户

- 需要快速理解长视频内容的学习者、研究者和内容创作者。
- 需要总结课程、访谈、播客、测评、操作演示视频的个人用户。
- 希望使用自有模型服务完成本地分析的技术用户。

### 3.2 核心场景

- 用户粘贴公开 Bilibili 视频链接，选择字幕优先模式，生成课程或访谈总结。
- 用户粘贴测评或操作演示视频链接，选择多模态模式，生成包含画面理解的报告。
- 用户在报告生成后继续追问，例如“这个视频的主要结论是什么”“作者推荐了哪些产品”“有哪些关键时间点”。
- 用户发现任务耗时过长或配置错误时，可以取消任务并重新配置分析。

## 4. 范围说明

### 4.1 一期必须支持

- NestJS 本地后端服务。
- 模型配置连接测试。
- 模型密钥安全存储或等价的本地安全处理机制。
- 分析任务创建、查询、取消。
- 字幕优先 + ASR 分析模式。
- 整段视频多模态分析模式。
- 快速、标准、深度三种分析深度。
- 公开 Bilibili 视频的字幕、音频、截图或视频素材处理。
- 结构化分析报告生成。
- 基于报告内容的继续追问。
- 后端媒体缓存和报告文件清理机制。

### 4.2 一期不支持

- 多用户账号体系和云端同步。
- Bilibili 登录态、Cookie 上传或扫码登录。
- 会员、付费、私密、地区受限或需要登录的视频。
- 自动切换分析模式或自动替换模型配置。
- 前端本地历史记录删除后自动删除后端所有资源。
- 浏览器内直接编辑报告内容。
- 团队协作、分享链接、公开发布报告。

## 5. 产品流程

### 5.1 模型准备流程

1. 用户在前端填写模型配置，包括配置名称、厂商、API Base URL、API Key、模型名称、模型能力、超时时间、最大并发数和是否默认。
2. 前端调用后端测试模型连接。
3. 后端使用用户提交的模型信息发起最小化测试请求。
4. 测试成功后，后端返回成功结果，并应具备保存密钥或后续任务可取用密钥的能力。
5. 测试失败时，后端返回可读错误原因，例如地址不可达、密钥无效、模型不存在、超时或响应格式不兼容。

### 5.2 创建分析任务流程

1. 用户读取视频信息并选择需要分析的分 P。
2. 用户选择分析模式、分析深度、是否生成关键截图、是否保留时间戳、输出语言、最大截图数和各能力模型配置。
3. 前端展示成本确认弹窗，用户确认后创建任务。
4. 后端校验请求参数、视频信息、分 P、模型能力和模型配置可用性。
5. 后端创建任务并返回任务快照。
6. 前端进入进度页并每 2 秒轮询任务状态。

### 5.3 任务执行流程

1. 后端将任务置为等待处理。
2. 后端根据模式执行视频信息确认、字幕获取、音频提取、ASR 转写、关键帧提取、模型分析和报告生成。
3. 后端持续更新任务状态、当前阶段说明和整体进度。
4. 任务完成后，后端生成报告 ID，并让任务进入完成状态。
5. 任务失败时，后端保留失败状态和可读失败原因。
6. 任务取消时，后端停止后续处理，并按清理策略释放临时资源。

### 5.4 报告与追问流程

1. 前端根据任务中的 `reportId` 请求报告详情。
2. 后端返回结构化报告。
3. 用户在报告页输入追问。
4. 后端基于当前报告、视频分析上下文和用户问题生成回答。
5. 后端返回一条助手消息，前端追加到当前会话中。

## 6. 功能需求

### 6.1 模型配置与连接测试

后端需要支持用户提交 OpenAI 兼容接口或常见模型厂商接口配置，并验证模型是否可用。

#### 6.1.1 输入字段

- 配置 ID，可选；编辑已有配置时使用。
- 配置名称。
- 模型厂商。
- API Base URL。
- API Key。
- 模型名称。
- 模型能力：文本总结、ASR 识别、图片理解、视频理解。
- 请求超时时间，单位为秒。
- 最大并发数。
- 是否设为该能力默认模型。

#### 6.1.2 验证规则

- API Base URL 必须是合法 URL。
- API Key 新增时必填；编辑时如果已经配置过密钥，可以留空表示不更换。
- 模型名称不能为空。
- 模型能力必须是 `text`、`asr`、`image`、`video` 之一。
- 请求超时时间不得小于 10 秒。
- 最大并发数不得小于 1，不得大于 20。

#### 6.1.3 后端行为

- 后端需要向指定模型服务发送最小化测试请求。
- 文本模型需要验证基础对话或文本生成能力。
- ASR 模型需要验证语音转写接口可达性；如果厂商不支持轻量测试，至少验证鉴权和模型可用性。
- 图片理解模型需要验证多模态输入能力。
- 视频理解模型需要验证厂商是否支持视频输入或可接受后端规划的视频片段输入方式。
- 后端不得在响应、日志、异常堆栈或报告中输出 API Key。
- 后端需要为后续分析任务提供按模型配置 ID 找到密钥和模型参数的能力；否则任务创建时只拿到模型配置 ID，无法完成真实模型调用。

#### 6.1.4 成功结果

连接测试成功时返回：

```json
{
  "ok": true
}
```

#### 6.1.5 失败结果

连接测试失败时返回统一错误结构，错误消息需要面向用户可理解：

```json
{
  "message": "模型连接测试失败，请检查 API Key、模型名称或服务地址。"
}
```

### 6.2 分析任务创建

后端需要支持创建异步分析任务。

#### 6.2.1 请求内容

任务创建请求包含：

- 视频信息。
- 分析选项。

视频信息包括：

- BV 号。
- 视频 URL。
- 标题。
- 封面 URL。
- UP 主名称。
- 发布时间。
- 视频总时长。
- 视频简介。
- 是否公开。
- 分 P 列表。

分析选项包括：

- 分析模式。
- 分析深度。
- 是否生成关键截图。
- 是否保留时间戳。
- 输出语言。
- 最大截图数量。
- 选择的分 P cid 列表。
- 各模型能力对应的模型配置 ID。

#### 6.2.2 校验规则

- BV 号不能为空。
- 视频 URL 必须是 Bilibili 视频地址。
- 视频必须标记为公开。
- 分 P 列表不能为空。
- 选择的分 P cid 必须存在于视频分 P 列表中。
- 字幕优先模式必须提供文本总结模型和 ASR 模型。
- 字幕优先模式开启关键截图时，必须额外提供图片理解模型。
- 整段视频多模态模式必须提供视频理解模型。
- 最大截图数量不得小于 0，不得大于 20。
- 输出语言不能为空，默认支持简体中文和 English。
- 分析深度必须是 `quick`、`standard`、`deep` 之一。

#### 6.2.3 创建成功

后端创建任务后立即返回任务快照，不等待分析完成。

任务初始状态为 `waiting`，进度为 0，当前阶段为等待处理。

### 6.3 任务状态管理

后端需要维护任务状态机，并让前端能够轮询最新状态。

#### 6.3.1 状态枚举

- `waiting`：等待处理。
- `fetching_video`：获取视频信息。
- `fetching_subtitle`：获取字幕。
- `extracting_audio`：提取音频。
- `transcribing`：语音转写。
- `extracting_frames`：提取关键帧。
- `analyzing`：模型分析。
- `generating_report`：生成报告。
- `completed`：已完成。
- `failed`：已失败。
- `cancelled`：已取消。

#### 6.3.2 进度要求

- 进度范围为 0 到 100。
- 任务完成时进度必须为 100。
- 任务失败或取消时进度保留最后一次有效值。
- 当前阶段说明需要能解释正在做什么，例如“正在提取第 2 个分 P 的音频”。

#### 6.3.3 状态查询

前端需要通过任务 ID 获取：

- 任务 ID。
- BV 号。
- 视频信息。
- 分析选项。
- 当前状态。
- 整体进度。
- 当前阶段说明。
- 错误信息。
- 创建时间。
- 完成时间。
- 报告 ID。

#### 6.3.4 异常要求

- 任务不存在时返回 404。
- 任务失败时必须写入 `errorMessage`。
- 外部模型调用失败时，需要区分超时、鉴权失败、限流、模型不存在和响应格式错误。
- Bilibili 资源不可访问时，需要返回明确原因。

### 6.4 任务取消

后端需要允许取消未完成任务。

#### 6.4.1 可取消状态

以下状态可以取消：

- `waiting`
- `fetching_video`
- `fetching_subtitle`
- `extracting_audio`
- `transcribing`
- `extracting_frames`
- `analyzing`
- `generating_report`

#### 6.4.2 取消行为

- 后端收到取消请求后，应尽快停止后续处理。
- 已经发出的外部模型请求如果无法中断，后端需要忽略其后续结果。
- 任务状态更新为 `cancelled`。
- 当前阶段说明更新为任务已取消。
- 后端根据清理策略删除临时视频、音频、字幕和截图缓存。
- 对已经完成、失败或取消的任务重复取消时，应返回当前任务状态，不应产生新的副作用。

### 6.5 字幕优先 + ASR 分析模式

该模式优先使用 Bilibili 公开字幕，在字幕缺失或不完整时使用 ASR 补全。

#### 6.5.1 处理要求

- 后端需要尝试获取视频公开字幕。
- 如果公开字幕存在，优先使用字幕作为文本基础。
- 如果字幕缺失，后端需要提取音频并调用 ASR 模型转写。
- 如果字幕覆盖不完整，后端可以使用 ASR 补足缺失片段。
- 开启关键截图时，后端需要提取关键帧并调用图片理解模型辅助分析。
- 关闭关键截图时，后端不得强制执行截图和图片理解步骤。

#### 6.5.2 适用内容

- 访谈。
- 课程。
- 演讲。
- 播客。
- 语言信息密集的视频。

### 6.6 整段视频多模态分析模式

该模式综合处理画面、字幕和音频，用于视觉信息密集的视频。

#### 6.6.1 处理要求

- 后端需要使用视频理解模型完成主要分析。
- 后端可以根据模型能力将视频切片、抽帧、提取音频或整理字幕后提交给模型。
- 后端需要控制单次请求大小，避免超过模型上下文或文件大小限制。
- 后端需要根据分析深度调整采样密度、分段数量和总结层级。

#### 6.6.2 适用内容

- 产品测评。
- 操作演示。
- 软件教程。
- 视觉信息密集的视频。

### 6.7 分析深度

后端需要根据分析深度调整处理成本和输出细节。

#### 6.7.1 快速分析

- 目标是尽快得到结论。
- 输出一句话结论、内容概览和基础章节。
- 减少截图、分段和模型调用次数。

#### 6.7.2 标准分析

- 目标是在速度和质量之间平衡。
- 输出完整章节、核心观点、事实案例和可信度提示。
- 适合作为默认分析深度。

#### 6.7.3 深度分析

- 目标是生成更全面的理解和洞察。
- 输出更细章节、更完整观点、更充分事实案例和更严谨可信度提示。
- 可以增加模型调用次数和中间分析步骤，但必须保留进度反馈。

### 6.8 结构化报告生成

后端需要生成可被前端直接展示的结构化报告。

#### 6.8.1 报告字段

报告需要包含：

- 报告 ID。
- 任务 ID。
- 视频信息。
- 创建时间。
- 一句话结论。
- 内容概览。
- 章节时间线。
- 核心观点。
- 重要事实和案例。
- 作者结论或立场。
- 关键截图。
- 推荐片段。
- 可信度提示。

#### 6.8.2 章节要求

每个章节需要包含：

- 章节标题。
- 开始时间，单位为秒。
- 章节摘要。

如果用户选择保留时间戳，章节和推荐片段必须尽量关联原视频时间点。

#### 6.8.3 核心观点要求

每个核心观点需要包含：

- 观点标题。
- 观点详情。

观点应来自视频内容，不得编造视频中不存在的信息。

#### 6.8.4 事实和案例要求

- 事实和案例应尽量来自视频原文、字幕、画面或明确的作者表达。
- 不确定内容不得写成确定事实。
- 对无法验证的信息，需要在可信度提示中说明。

#### 6.8.5 关键截图要求

开启截图时，截图项需要包含：

- 图片 URL。
- 时间戳，单位为秒。
- 图片描述。

关闭截图时，截图列表可以为空。

### 6.9 报告追问

后端需要支持用户围绕当前报告继续提问。

#### 6.9.1 输入

追问请求包含：

- 问题文本。

#### 6.9.2 校验

- 问题不能为空。
- 问题长度需要设置合理上限。
- 报告必须存在。

#### 6.9.3 回答要求

- 回答必须基于当前报告和后端保留的分析上下文。
- 如果报告中没有相关信息，需要明确说明无法从当前视频确认。
- 回答不应泄露模型配置、API Key、服务端路径或内部异常堆栈。

#### 6.9.4 返回结构

后端返回一条助手消息，包含：

- 消息 ID。
- 角色，固定为 `assistant`。
- 文本内容。
- 创建时间。

### 6.10 资源缓存与清理

后端需要管理分析过程中产生的临时文件和报告资源。

#### 6.10.1 临时资源

可能产生的临时资源包括：

- 视频片段。
- 音频文件。
- 字幕文件。
- 截图文件。
- 模型中间结果。

#### 6.10.2 清理要求

- 任务取消后应清理未被报告引用的临时资源。
- 任务失败后应清理可安全删除的临时资源，并保留必要错误诊断信息。
- 已完成任务的报告和截图应保留，供前端读取。
- 后端需要提供周期性清理机制，避免本地磁盘长期增长。
- 清理策略不得依赖浏览器 localStorage 是否删除历史记录。

### 6.11 本地历史边界

当前前端使用浏览器 localStorage 保存任务快照和历史记录索引。后端不需要直接管理前端历史列表，但需要保证：

- 已完成任务可以通过任务 ID 或报告 ID 被读取。
- 前端删除本地历史记录不会自动删除后端报告和缓存。
- 后端资源清理由后端清理策略负责。

## 7. API 需求

### 7.1 测试模型连接

```http
POST /api/model-configs/test
Content-Type: application/json
```

请求体为模型配置草稿。

成功响应：

```json
{
  "ok": true
}
```

失败响应：

```json
{
  "message": "模型连接测试失败，请检查 API Key、模型名称或服务地址。"
}
```

### 7.2 创建分析任务

```http
POST /api/analysis/jobs
Content-Type: application/json
```

请求体：

```json
{
  "video": {
    "bvid": "BV1xxxxxxxxx",
    "url": "https://www.bilibili.com/video/BV1xxxxxxxxx",
    "title": "视频标题",
    "coverUrl": "https://example.com/cover.jpg",
    "ownerName": "UP 主",
    "publishedAt": "2026/6/16 20:00:00",
    "duration": 3600,
    "description": "视频简介",
    "isPublic": true,
    "parts": [
      {
        "cid": 123,
        "page": 1,
        "title": "P1",
        "duration": 3600
      }
    ]
  },
  "options": {
    "mode": "subtitle",
    "depth": "standard",
    "generateScreenshots": true,
    "keepTimestamps": true,
    "outputLanguage": "简体中文",
    "maxScreenshots": 12,
    "selectedPartCids": [123],
    "modelProfileIds": {
      "text": "model-text-id",
      "asr": "model-asr-id",
      "image": "model-image-id"
    }
  }
}
```

成功响应为分析任务对象。

### 7.3 查询任务

```http
GET /api/analysis/jobs/:id
```

成功响应为分析任务对象。

### 7.4 取消任务

```http
POST /api/analysis/jobs/:id/cancel
```

成功响应为分析任务对象。

### 7.5 获取报告

```http
GET /api/analysis/reports/:id
```

成功响应为结构化分析报告。

### 7.6 报告追问

## 8. 数据结构需求

### 8.1 AnalysisJob

分析任务需要包含：

- `id`：任务 ID。
- `bvid`：视频 BV 号。
- `video`：视频信息。
- `options`：分析选项。
- `status`：任务状态。
- `progress`：进度百分比。
- `currentStage`：当前阶段说明。
- `errorMessage`：失败原因，可选。
- `createdAt`：创建时间。
- `completedAt`：完成时间，可选。
- `reportId`：报告 ID，可选。

### 8.2 AnalysisReport

分析报告需要包含：

- `id`：报告 ID。
- `jobId`：任务 ID。
- `video`：视频信息。
- `createdAt`：创建时间。
- `summary`：一句话结论。
- `overview`：内容概览。
- `chapters`：章节时间线。
- `keyPoints`：核心观点。
- `facts`：事实和案例。
- `conclusion`：作者结论或立场。
- `screenshots`：关键截图。
- `recommendedSegments`：推荐片段。
- `confidenceNotes`：可信度提示。

## 9. 非功能需求

### 9.1 技术栈

- 后端框架采用 NestJS。
- 语言采用 TypeScript。
- 接口返回 JSON。
- 后端服务需要支持前端以相对路径 `/api` 访问。

### 9.2 本地优先

- 服务默认面向本地单机运行。
- 不要求公网部署能力。
- 不要求多用户隔离。
- 本地文件、缓存和报告路径不得直接暴露给前端。

### 9.3 安全

- API Key 不得写入浏览器本地存储。
- API Key 不得出现在接口响应中。
- API Key 不得出现在普通运行日志中。
- API Key 不得出现在错误堆栈、任务错误原因或报告内容中。
- 后端需要对外部 URL 请求设置超时和大小限制。

### 9.4 稳定性

- 长任务执行过程中，前端轮询不应导致重复执行任务。
- 服务重启后，应尽量保留已完成报告和可查询任务记录。
- 对外部模型和 Bilibili 接口的失败，需要有超时、重试或明确失败状态。
- 模型限流时，后端需要返回可理解错误，不应让任务无限等待。

### 9.5 成本控制

- 后端不得在用户未确认的情况下自动切换分析模式。
- 后端不得在用户未选择视频理解模型时自动升级为多模态分析。
- 深度分析可以增加模型调用次数，但必须受任务配置和模型并发限制约束。
- 关闭截图时，不应调用图片理解模型。

### 9.6 可观测性

- 后端需要记录任务开始、阶段变化、完成、失败和取消事件。
- 日志需要隐藏密钥和敏感请求头。
- 错误日志可以记录内部诊断信息，但返回给前端的错误必须面向用户可读。

## 10. 错误与提示需求

后端错误消息需要帮助用户判断下一步操作。

常见错误包括：

- 模型服务无法连接。
- API Key 无效。
- 模型名称不存在。
- 模型能力与当前分析模式不匹配。
- Bilibili 视频不可访问。
- 未找到公开字幕。
- 音频提取失败。
- ASR 转写失败。
- 截图提取失败。
- 模型分析超时。
- 报告生成失败。
- 任务不存在。
- 报告不存在。

错误响应不应暴露内部文件路径、依赖库堆栈、完整外部请求内容或 API Key。

## 11. 验收标准

### 11.1 模型连接测试

- 使用有效模型配置时，接口返回 `{ "ok": true }`。
- 使用错误 API Key 时，接口返回可读错误。
- 使用错误模型名称时，接口返回可读错误。
- 后端日志和响应中不出现 API Key 明文。

### 11.2 创建任务

- 字幕优先模式选择文本模型和 ASR 模型后，可以成功创建任务。
- 字幕优先模式开启截图且未选择图片模型时，后端拒绝创建任务。
- 多模态模式未选择视频模型时，后端拒绝创建任务。
- 未选择任何分 P 时，后端拒绝创建任务。
- 创建成功后立即返回任务对象，状态为 `waiting`。

### 11.3 任务进度

- 前端轮询任务接口时，可以看到状态和进度变化。
- 任务完成后状态为 `completed`，进度为 100，并返回 `reportId`。
- 任务失败后状态为 `failed`，并返回 `errorMessage`。
- 任务取消后状态为 `cancelled`，后续轮询不再推进任务。

### 11.4 报告生成

- 完成任务可以通过报告 ID 获取报告。
- 报告包含一句话结论、内容概览、章节、核心观点、事实案例和可信度提示。
- 开启时间戳时，章节和推荐片段包含可用于跳转的秒数。
- 关闭截图时，报告截图列表可以为空。
- 开启截图时，截图 URL 可以被前端正常展示。

### 11.5 报告追问

- 对存在的报告提交有效问题，可以返回助手回答。
- 空问题会被拒绝。
- 不存在的报告 ID 返回 404。
- 回答基于当前报告内容，不编造无法从视频确认的信息。

### 11.6 清理与安全

- 取消任务后，临时媒体资源被清理或进入待清理队列。
- 失败任务不会遗留无限增长的临时文件。
- 已完成报告可以在清理临时文件后继续读取。
- 接口响应、日志和报告均不泄露 API Key。

## 12. 联调注意事项

- 当前前端创建分析任务时只传 `modelProfileIds`，不会把 API Key 放进任务请求。
- 后端必须能根据模型配置 ID 获取对应密钥和模型参数，或者补齐模型配置保存接口并让前端在测试成功后调用。
- 如果只实现 `/api/model-configs/test` 而不保存密钥，分析任务将无法真实调用用户选择的模型。
- 前端每 2 秒轮询任务状态，后端接口需要支持频繁读取且不触发重复执行。
- 前端历史记录保存在浏览器 localStorage，后端不要依赖该数据作为任务唯一来源。
